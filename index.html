    Introducing Lil'JS

    Lil'JS aims to be a general purpose JavaScript collection of modules for common JavaScript core development
    Modules are designed keeping in mind modularity, lightweight, independence, low-coupling and reusability

    Tiny

    Tiny

    Modules are designed keeping size in mind, trying to create them as lightweight as possible. Usually they are less than 1KB (gzipped)

    Modular

    Modular

    Each module has it's own resposability and no coupling with others modules

    Support

    Cross-engine

    Runs in multiple JavaScript engines, including browsers, node.js and any other ES5 compliant engine

    Design Principles & Features

    Single responsability

    Each module was designed according to the single module responsability principle

    Low-coupling

    Modules arises from the idea to be much independent as possible, avoiding dependency between them

    Well tested

    Each module has a high test coverage and pass tests in node.js, headless and real browsers

    Lightweight

    Modules are too tiny and lightweight as possible. Only necessary code will be put in

    CommonJS

    It uses CommonJS module definition pattern to support multiple JavaScript environments

    Well documented

    Each module has it's own basic usage, examples and API documentation, based on Markdown
